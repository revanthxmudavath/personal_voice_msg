# T03 — SQLite schema and delivery state machine

## Scope

- Versioned migration for sources, inspiration cards, messages, runs, audio
  artifacts, and deliveries.
- Foreign-key-enabled operational connections and WAL mode.
- Explicit content and delivery state transitions.
- Atomic reservation keyed by opaque recipient identity and Pacific date.
- Persisted state that survives closed/reopened database instances.

## Red evidence

Both subagent-owned suites were run before implementation:

```text
tests/fast/test_database_migrations.py
ModuleNotFoundError: No module named 'personal_voice_msg.database'

tests/fast/test_delivery_state_machine.py
ModuleNotFoundError: No module named 'personal_voice_msg.database'
```

Independent review added red regressions for unknown future migration versions,
raw phone numbers used as recipient keys, and datetimes used as Pacific dates.
Deeper probes added unknown lower versions, digits-only phone keys, and drifted
version-1 schemas. All failed against the earlier implementations before the
boundaries were tightened. A final red fixture preserved every expected column
and index name while weakening delivery constraints; exact schema-definition
validation now rejects it.
