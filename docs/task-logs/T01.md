# T01 — Repository and no-mock test foundation

## Scope

- Minimal Python 3.12 package managed by `uv` in a project-local `.venv`.
- Fast repository-policy tests for mocks, lockfile freshness, planted credentials,
  and GitHub workflow YAML.
- Ruff, mypy, pytest markers, concise development documentation, and CI.
- No production application behavior.

## Red evidence

Command:

```powershell
uv run pytest tests/fast/test_repository_policy.py -q
```

Result: 9 failed because `scripts/repository_policy.py` did not exist. The
violation cases additionally required useful diagnostics for the planted file
or invalid state. Independent review later added five red cases for direct
`pytest.MonkeyPatch` use and missing or structurally invalid workflows; all five
failed against the initial checker before it was tightened.

## Green evidence

```text
uv sync --locked
Resolved 15 packages; audited 15 packages.

uv run pytest -m fast
14 passed.

uv run ruff check .
All checks passed.

uv run mypy src
Success: no issues found in 1 source file.

uv run python scripts/repository_policy.py all --root .
Exit code 0.
```

The tests use real temporary directories, real subprocesses, and real `uv`
lockfile checks. They do not use mocks or monkeypatching.

## Audit hardening on 2026-07-21

An independent T01-T05 audit reproduced policy bypasses that the earlier scan
did not cover. Ten new red cases proved that syntax-invalid or non-UTF-8 Python
was silently skipped, dynamic `pytest.MonkeyPatch`/`unittest.mock` imports were
not detected, and sensitive voice, session, recipient, token, and private-key
artifacts could pass the secret scan.

The policy now:

- fails closed when a Python file cannot be parsed or decoded;
- detects the bounded dynamic mock-access forms demonstrated by the audit;
- rejects known recipient, WAHA token/session, voice-embedding, and private-key
  artifact names before attempting text decoding;
- rejects private-key markers in otherwise ordinary text files; and
- permits explicitly named `.example` documentation fixtures.

Focused T01/T02 verification after the fixes: `60 passed`. Ruff, strict mypy,
the lockfile check, and the full repository-policy command were clean.

Static policy cannot prove the absence of every runtime-generated import or an
unknown provider-token format. New integration code remains subject to review,
and provider-specific token patterns must be added when T15 selects the final
WAHA credential format. Git-history and image/artifact scanning remain part of
the T18 hardening gate.
